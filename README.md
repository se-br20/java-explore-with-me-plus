## Commets
### Доп. фича:
- коментарии к событиям
- модерация коментариев со стороны админов
- сохранение информации о том, кто был модератором
- получение комментов с пагинацией (постраничной, а не offset и limit как в основном сервисе)
#### Интеграция в существующий сервис
- В DTO-сущности добавлено поле commentsCount.
- В методах сервиса добавлена подгрузка к DTO-сущностям событий числа комментариев одним запросом.
- При запросах к эндпоинтам событий, везде где возвращается EventFullDto или EventShortDtо или возвращаются коллекции, содержащие эти сущности, так же возвращается и соответсвующее поле commentsCount с количеством комментариев у этого события.



### Публичные эндпоинты

**GET /events/{eventId}/comments**
- Получение списка опубликованных комментариев к событию.
- Параметры: `eventId` (path), `from`, `size` (query, опционально).
- Ограничения: возвращаются только комментарии со статусом `APPROVED`. Пагинация: `from` – смещение страницы, `size` – количество на странице (по умолчанию `from=0`, `size=10`).

**GET /comments/{commentId}**
- Получение конкретного комментария по его идентификатору.
- Параметры: `commentId` (path).
- Ограничения: комментарий должен быть со статусом `APPROVED`, иначе возвращается `404 Not Found`.

### Приватные эндпоинты

**POST /users/{userId}/comments/events/{eventId}**
- Создание нового комментария.
- Параметры: `userId`, `eventId` (path).
- Тело запроса (обязательно): `NewCommentDto`.
- Ограничения: пользователь с указанным `userId` должен существовать; событие должно быть опубликовано. Новый комментарий получает статус `PENDING`.

**PATCH /users/{userId}/comments/{commentId}**
- Обновление своего комментария (текст или статус).
- Параметры: `userId`, `commentId` (path).
- Тело запроса (обязательно): `UpdateCommentUserRequest`.
- Ограничения: пользователь должен быть автором комментария.
    * Изменение текста разрешено только если комментарий находится в статусе `PENDING`.
    * Статус можно изменить только на `DELETED`. Удаление возможно независимо от текущего статуса.

**DELETE /users/{userId}/comments/{commentId}**
- Удаление своего комментария.
- Параметры: `userId`, `commentId` (path).
- Ограничения: пользователь должен быть автором. Комментарий получает статус `DELETED`.

### Административные эндпоинты

**GET /admin/comments**
- Получение комментариев для модерации (со статусом `PENDING`).
- Параметры: `from`, `size` (пагинация).

**PATCH /admin/comments/{userId}/{commentId}**
- Модерация комментария – изменение текста и/или статуса.
- Параметры: `userId` (path, идентификатор модератора), `commentId` (path).
- Тело запроса (обязательно): `UpdateCommentAdminRequest`.
- Ограничения: доступ только администраторам. Можно установить любой статус (`APPROVED`, `REJECTED`, `DELETED`) и изменить текст. Указанный `userId` сохраняется как модератор.

**DELETE /admin/comments/{userId}/{commentId}**
- Удаление любого комментария администратором.
- Параметры: `userId` (path, идентификатор модератора), `commentId` (path).
- Ограничения: доступ только администраторам. Комментарий переводится в статус `DELETED`, указанный `userId` сохраняется как модератор.
